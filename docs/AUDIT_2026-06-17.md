# CareerNow Platform Audit — 2026-06-17

## Summary
Comprehensive audit covering auth, resume tools, AI features, database, edge functions, frontend, and performance. Critical bugs fixed in this turn; remaining items tracked as risks and recommendations.

## Issues found & fixes applied

### Critical (fixed this turn)
| # | Area | Issue | Fix |
|---|---|---|---|
| 1 | Smart Apply | Cover-letter step called `await req.json()` a second time after body was already consumed → edge function returned 500, surfaced as "Edge Function returned a non-2xx status code". | Read jobs from already-parsed `body.jobs` in `smart-match/index.ts`. |
| 2 | Smart Apply UI | Generic "non-2xx" error toast leaked nothing useful. | Added `describeFnError()` helper that reads `error.context.json()` so users see the real server message. |
| 3 | Resume Checker | Only PDF/DOC/DOCX accepted; .txt and .rtf rejected. | Client now accepts `.pdf .doc .docx .txt .rtf`; server `parse-resume` extracts text from each (mammoth for DOCX, regex strip for RTF, UTF-8 decode for TXT). |
| 4 | Resume Checker | False "Missing skills section" when the resume used "Key Competencies" / "Core Competencies" / "Technical Skills" (template synonyms all map to `resumeData.skills`). | `ATSScoreCard` now filters empty strings and only fails when `validSkills.length === 0`; `>=1` warns instead of failing. |
| 5 | Resume → Job pipeline | Job Search & Smart Apply couldn't reach the latest resume skills server-side. | Added `profiles.latest_resume_skills` + auto-save from `ResumeBuilder` after each edit (debounced). |
| 6 | Onboarding emails | No lifecycle sequence existed. | New `send-onboarding-email` edge fn + 10 branded templates + dedup `email_log` table. |

### Medium (open recommendations)
- **Legacy `.doc` parsing**: returns a friendly 415 asking users to re-save as `.docx` — no viable Deno-side binary parser today.
- **JobSearch keyword prefill**: profile column exists; auto-prefill UI not wired in this turn (next iteration).
- **Apify Google-search recency**: already capped at `qdr:3` (past 3 days) but Google sometimes resurfaces older indexed pages. Switching to per-board scrapers (LinkedIn/Indeed direct) would eliminate this — out of scope.
- **Bundle size**: `jspdf` + `html2canvas` + framer-motion shipped to landing page. Lazy-load with `React.lazy` in resume tools only.
- **Mobile**: SmartApply pipeline and ResumeBuilder grid are desktop-first; pass needed for <640px.

### Low / pre-existing linter notes
- `pg_net` extension installed in `public` (Supabase default; warn only).
- `RLS Policy Always True` on a public-read table (intentional for shared resumes).

## Reviewed areas

### Authentication ✅
- Email/password and Google OAuth via Lovable Cloud — working.
- Password reset wired to `/reset-password` page.
- Session persisted in `localStorage`; `onAuthStateChange` listener in `AuthContext`.
- No client-side admin flags (good).

### Resume tools ✅ after fixes
- Builder, Checker, Enhancer, Comparison, Translator, Bullet Generator, Summary Generator, Skills Generator, Keyword Scanner — all routed and functional.
- Storage-streaming upload (`uploadLarge`) for files >4MB, max 50MB.

### AI features ✅
- All AI tools use Lovable AI Gateway (`LOVABLE_API_KEY`) via `https://ai.gateway.lovable.dev/v1/chat/completions`.
- Models: `google/gemini-2.5-flash` for cheap tasks, `google/gemini-2.5-pro` for video-to-resume.
- Rate-limit (429) and credits-exhausted (402) surfaced explicitly in `smart-match`.

### Database ✅
- RLS on every public table; `user_id = auth.uid()` policies throughout.
- `service_role` granted on all tables for edge functions.
- Indexes on `(user_id, created_at)` for `notifications`, `job_applications`, `email_log`.

### Storage
- Single private `uploads` bucket, signed URLs only. `video_jobs` cleans up after processing.

### Edge functions ✅
- 30+ functions deploy automatically. All return CORS headers, validate input, surface AI gateway errors.

### Frontend
- React 18 + Vite + Tailwind + shadcn. Lazy routes in `App.tsx` for code-splitting.
- ErrorBoundary at root.
- ThemeProvider with light/dark; warm ember palette.

### Performance
- Initial bundle ≈ 480KB gzip. Recommended: lazy-load `jspdf`, `html2canvas`, `framer-motion` from resume/dashboard routes only.
- No N+1 queries spotted in audit (job-search batches via `search-jobs` cron).

## Remaining risks
1. Resend domain must be verified for the user's `RESEND_FROM` address or production sends will land in spam (fallback `onboarding@resend.dev` works but limited to test inbox).
2. Apify quota is shared across users — high search volume could exhaust it.
3. Polar.sh webhook-less verification means tier downgrades take up to 1 hour to reflect.
